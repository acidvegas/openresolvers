## ūüďĖ Overview

This repository maintains a curated list of public DNS resolvers that have been verified to be stable, responsive, and returning validated results. The resolver list is updated every 24 hours through an automated pipeline.

## ūüĒ¨ Methodology

Our validation process combines multiple techniques to ensure only reliable resolvers make it into the list:

- ūüĒó **Aggregation** ‚ÄĒ Resolvers are pulled from over 100 different public sources and merged into a single deduplicated list.
- ūüď° **Port Scanning** ‚ÄĒ We continuously scan for open port 53 (DNS) across the IPv4 space 24/7 to discover new resolvers as they come online.
- ‚ŹĪÔłŹ **Stability Testing** ‚ÄĒ Each discovered resolver is tested repeatedly over time. Only resolvers that demonstrate consistent uptime and response times are included.
- ‚úÖ **Response Validation** ‚ÄĒ Resolvers must return correct, non-poisoned responses for a set of known domains. Any resolver returning invalid or manipulated results is removed.
- ūüĒĄ **Daily Refresh** ‚ÄĒ The entire list is regenerated daily, dropping any resolvers that have gone offline or started misbehaving.

## ūüĒí Privacy-Focused Resolvers

For users who prefer well-known, privacy-respecting DNS providers, we also maintain a separate [`privacy-resolvers.txt`](privacy-resolvers.txt) file. This includes providers like Cloudflare, Quad9, Mullvad, AdGuard, NextDNS, and others that have strong no-logging policies and support encrypted DNS protocols (DoH/DoT).

## ūüďĄ Format

The `resolvers.txt` file contains one IPv4 address per line ‚ÄĒ no ports, no comments, no extra formatting. Ready to use with tools like `massdns`, `dnsx`, `puredns`, `shuffledns`, etc.
